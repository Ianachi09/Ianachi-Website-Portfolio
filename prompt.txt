**Improvement**
### 1. The Navigation Redundancy (UX Flaw)

You mentioned having a Top Nav, but also a Sidebar that is "always shows in desktop" (with an off-canvas style).

- **The Flaw:** "Off-canvas" strictly means the menu lives outside the viewport and must be toggled (via a hamburger button) to slide in. If it _always_ shows on desktop, it is a fixed sidebar, not off-canvas. Furthermore, if you have a Top Nav _and_ a fixed Sidebar on desktop, you are eating up a lot of screen real estate for redundant navigation.
    
- **The Correction:** Decide on one primary navigation method for desktop. The cleanest approach for your "center-focused forum" vibe is to keep the Top Nav for your main links on desktop, and use the Sidebar purely as a toggleable, off-canvas element for mobile users (holding the profile and mobile nav links).
    

### 2. Live Fetching from GitHub (Architecture Flaw)

Your plan to pull Project and About data directly from your GitHub repository (via `.md` files in a `/web_src` folder) is a great way to handle maintenance without touching HTML every time you finish a new programming or university assignment.

- **The Flaw:** If you try to fetch this data dynamically using JavaScript (client-side) every time a user loads the page, you will quickly hit the GitHub API's strict rate limits. Your website will break and fail to load content for visitors once that limit is reached.
    
- **The Correction:** You need to use a Static Site Generator (SSG) like Astro, Next.js, or Hugo. These tools fetch your GitHub data at _build time_ (when you deploy the site), compile it into standard HTML, and serve that to the user. This guarantees the site loads instantly and never hits API rate limits during normal traffic.
    

### 3. The Cinnamoroll Terminal Design

Designing a terminal interface on the right side of your Hero section will feel right at home with configuring environments like Hyprland or customizing Waybar—you are just porting that skill to the web.

- **Design Tip:** To prevent the terminal from clashing with the fluffy, white/blue Cinnamoroll theme, avoid a harsh black background. Instead, utilize a soft, pastel color palette (similar to Catppuccin Latte or Macchiato) for the terminal window. Use light blues for the syntax highlighting to keep the whole page cohesive.
    

### 4. Single-Column Project Cards

Displaying your repositories as single, center-focused cards with gaps is a great way to keep the user's attention on one project at a time.

- **The Flaw:** On a large, ultra-wide desktop monitor, a single column will stretch the cards horizontally, making text lines painfully long to read and making small GIFs look awkwardly tiny inside a massive box.
    
- **The Correction:** You must apply a strict `max-width` (e.g., `max-width: 800px;`) to the container holding your project cards. This ensures that even on a 4K monitor, the cards remain a readable, compact size floating perfectly in the center of the grid background.
    

### 5. Contact Form Handling

- **The Flaw:** HTML forms cannot send emails by themselves. If your site is purely frontend, the submit button won't do anything.
    
- **The Correction:** You will need to route your form through a free backend service like Formspree or Web3Forms. You just paste their provided URL into your HTML `<form action="...">` attribute, and they handle routing the submission to your email inbox.
